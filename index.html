Ejercicio 1 HTML

Ejercicio análisis de un documento html con las herramientas del navegador Chrome.

Analice la página web del sitio www.frc.utn.edu.ar con Chrome devtools

Responda las siguientes preguntas:

1) ¿Cuantos Kilobytes se transfieren al navegador para visualizar la página de manera completa? Cuanto tiempo demora en cargar la página. Si se utiliza el caché del navegador, demora lo mismo?

Se transfieren 1.4 MB y demora 4.5 seg en cargar, pero con el cache activado demora 3.66 seg.

2) ¿La página posee errores o warnings?

No, la pagina no posee ningun error, con la excepcion de que al usar Adblock salta el error de carga de pblicidad "Failed to load resource: net::ERR_BLOCKED_BY_CLIENT"

3) ¿Cuál es el archivo que demora mas tiempo en descargarse de la página

El documento de www.frc.utn.edu.ar es el que mas trada con una duracion de 3 seg.

4) ¿Todos los archivos provienen del mismo servidor?

No, algunos vienen de Youtube por ejemplo

5) ¿Cuántas imágenes posee?, qué tiempo que demora en descargarlas? ¿Cual es el imagen de mayor tamaño en kilobytes que se descarga del sitio?

Contiene 69 imagenes, tarda 3.96 seg en cargarlas sin cache, y la mas grande es la imagen logo de CISCO ORACLE con un tamaño de 20.5 kB

6) ¿Cuántos archivos de hojas de estilo utiliza?

Utiliza 4 archivos de StyleSheet

7) En qué versión de html indica el documento que está realizada la página

Esta usalndo XHTML 1.0 Transitional

8) ¿Los archivos javascript y/o css están minificados?

Si, porque tienen el .min

9) ¿Qué tiempo demoraría la página en descargar en un equipo con señal de datos de una red Regular 3G?

Con la opcion de Fast 3G y el cache desactivado tarda 10.55 seg, mientras que con la opcion de Slow 3G tarda 35.83 seg

10) La página es compatible con el estándar html indicado en la cabecera. Analice la misma en https://validator.w3.org/

No, no es compatible y se detectan 23 errores

11) Analice las recomendaciones para mejorar la velocidad de carga de la página en https://developers.google.com/speed/pagespeed/

Se pueden observar recomendaciones como reducir el contenido JavaScript que no se usa y reducir el tiempo de respuesta inicial del servidor

12) Realice el mismo análisis con el sitio www.cba.gob.ar

La pagina del gobierno presenta un peor rendimiento en cuanto a velocidad de carga, y algunas de las recomendaciones obtenidas son habilitar la compresion de texto o reducir la cantidad de contenido de css que no se usa.
